To build a production quality platform, where the key application in the platform is to perform Pre Trade Control checks. 2. This application will be called "ArT" and will be supported by stubs (supporting input and output processing systems / apps, which we are not going to build out the full logic in Phase 1). 3. The role of a stub (call it "Stub 1") for example will be to randomly generate trades with trade economics that trigger Reg T and FINRA 4210 checks and receive, persist and render the pre-trade control app's responses. The role of another stub (call it "Stub 2") will be to handle the supervisory approvals required for one of ArT's responses e.g. Hard Block Override. 4. This platform should have the entire tech stack built out to do pre trade control checks to adhere to Regulation T and FINRA 4210 regulatory 2026 updates, for both Equities and Fixed Income products. 5. It should have an UI feature to enable users to specify the rules, update and maintain them in Natural Language. 6. It should also have a UI feature to enable users to generate reports queried in Natural Language. The above will be updated on the go as we discover more goals through the life of the project in building out the platform.
